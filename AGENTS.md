# Session Continuation Rule

- Unless the user explicitly says to stop, pause, or switch focus, continue executing the next meaningful product-improvement wave in the same session.
- Do not stop after a single completed wave just to ask for confirmation when a clear next step exists.
- Only stop the active session when:
  - a destructive or materially branching user decision is required,
  - credentials, external access, or authority are missing,
  - or fresh verification shows a blocker with no reasonable recovery path.
- When one wave completes, immediately:
  1. gather fresh verification evidence,
  2. select the next highest-value wave,
  3. continue implementation.
